# 5G-A 异构跨域信任底座：全景架构与技术演进路线图 (Architecture & Roadmap)

## 1. 关键科学问题与研究目标 (Scientific Problems & Objectives)
针对 5G-A 跨域数据流通与公专混合组网场景，传统中心化 PKI 信任体系暴露出身份认证跨域难、证书互认效率遭遇瓶颈、以及通感数据交互溯源缺失等核心矛盾。
本项目旨在攻克**“面向异构网络的分布式信任模型技术”**，构建一个分布式、弹性可扩展的信任协作底座。以“跨域互认、高效协同”为核心，通过异构双层区块链与去中心化预言机的深度融合，实现全域证书状态的实时同步、通感数据的抗篡改溯源，为 5G-A 行业应用提供高扩展、低时延的信任服务底座。

---

## 2. 总体架构设计：双层区块链与跨域总线 (Dual-Layer Architecture)

为解决信任服务低效与扩展性不足的矛盾，系统采用物理隔离与逻辑协同的 L0/L1 双层区块链架构：

### 2.1 L0 信任基座层 (Trust Anchor Layer)
* **物理选型**: Hyperledger Fabric
* **网络定性**: 封闭网络场景（模拟 5G-A 核心网）。
* **核心特性**:
    * **DPKI 证书互认体系**: 将多个 CA 机构与 Fabric 共识节点（Orderer/Peer）进行一对一物理连接，构建跨域证书互认体系。
    * **全生命周期管理**: 通过底层链码（Chaincode）实现数字证书签发、吊销、验证的链上自动化存证，彻底替代传统 PKI 依赖的中心化信任锚点。

### 2.2 L1 业务执行层 (Business Execution Layer)
* **物理选型**: FISCO BCOS (多群组架构)
* **网络定性**: 容忍少数恶意节点的半开放联盟场景或开放环境（模拟 5G-A 边缘计算与公专混合接入区）。
* **核心特性**:
    * **高并发架构**: 引入 FISCO BCOS 的 DAG (有向无环图) 并行处理引擎，支持交易异步提交与多通道并发确认，突破传统链式结构单区块打包的效率瓶颈。
    * **信任凭据协议**: 规范数字证书、远程证明基线值、分布式数字身份（DID）的标准化上链格式与验证逻辑。

### 2.3 异构跨域可信总线 (Cross-Domain Trust Bus)
* **物理选型**: Chainlink 去中心化预言机网络 + 专属 External Adapter集群
* **核心特性**: 作为 L0 与 L1 之间的状态机信使，提供物理级的数据穿透与事件驱动能力，消除跨域网关的单点故障。

## 3. 核心机制设计与课题任务映射 (Core Mechanisms)

### 3.1 任务 2-1：基于场景的共识算法选配引擎 (Consensus BaaS Engine)
在链启动部署阶段，系统提供统一的配置中心，根据 5G-A 目标场景特性自动化选配共识算法：
1.  **Raft 共识**: 部署于 L0 层（Fabric），适用于没有恶意节点的封闭网络，对吞吐量要求不高的小规模核心场景。
2.  **PBFT 共识**: 部署于 L1 层的部分群组，适用于半开放的联盟场景，通过流程优化实现多 CA 协同的高性能节点共识。
3.  **DPoS (rPBFT) 共识**: 部署于 L1 层的边缘群组，适用于开放环境下、要求极高并发与低延迟，且对去中心化程度要求适中的海量通感数据交互场景。

### 3.2 任务 2-2：网元证书跨域管理机制 (DPKI State Synchronization)
* **痛点消除**: 传统 PKI 依赖冗长证书链和 CRL/OCSP 协议，存在严重延迟和单点故障。
* **系统实现**: 将网元证书的生命周期状态实时固化在 L0 账本中。当发生跨域验证时，L1 智能合约无需依赖传统交叉认证，而是抛出协同鉴权事件。预言机实时穿透至 L0 检索证书状态（颁发机构、有效期、吊销状态），并通过多方见证将其物理回写至 L1，实现全域证书状态的毫秒级实时同步。

### 3.3 任务 2-3：通感数据全链路溯源模型 (Dual-Stage Traceability)
基于预言机的异步回调机制，构建严格的“请求-响应”双阶段存证流：
1.  **阶段一 (请求上链)**: 跨域数据请求发生时，源域生成包含 `[请求方DID]`, `[目标域标识]`, `[数据指纹]` 的交互指令，经双方签名后在 L1 层上链，生成 `RequestID`。
2.  **阶段二 (响应追加)**: 预言机调度网关校验权限并获取数据。数据提供方在响应时，通过预言机将返回数据的哈希值及相关授权记录，基于相同的 `RequestID` 追加写入 L1 层的同一事务链上。
* **效果**: 通过链式关联所有跨域操作日志，确保通感数据从采集、传输到调用的端到端 100% 可审计性。

### 3.4 任务 1-1/1-2：5G-A 原生安全架构与威胁模型应对 (Security Architecture)
* **跨域可信与隔离**: 设计严格的信任域划分方案，根据网络隔离策略与数据敏感度，在 L1 层划分多个逻辑信任域（对应 FISCO BCOS 的 Group），域内维护独立的 DID 与访问控制策略。
* **内生免疫与外挂防护协同**: 系统后端采用 Go 语言构建的监听中枢与回写中枢，具备对底层 EVM ABI 异常编码的自适应防死循环能力（内生免疫）；同时预言机网关充当外挂防护层，隔离异构协议栈的直接攻击。

## 4. 工程研发路线图 (Engineering Roadmap & TODO List)

为将上述架构落地为可验证的代码底座，整体研发划分为四个阶段：

### Phase 1: 业务语义重构与领域驱动设计 (Domain-Driven Design)
* [ ] **前端重构**: 将现有大屏终端 UI 彻底改造为“5G-A 网元跨域协同平台”。文案术语全面对齐 5G-A 通感数据、DID、DPKI 等专业名词。
* [ ] **数据结构规范化**: 在代码层统一定义跨域请求指令的 JSON Schema，包含 `sourceDID`, `targetDomain`, `dataFingerprint`, `timestamp` 等标准字段。

### Phase 2: L0 层 DPKI 核心链码开发 (L0 Chaincode Implementation)
* [ ] 在 Fabric 编写 `DPKI_Registry` 智能合约。
* [ ] 实现 CA 机构一对一绑定的初始化逻辑。
* [ ] 实现核心接口：`RegisterNetworkElement(DID, PubKey)`, `RevokeCert(DID)`, `GetCertStatus(DID)`。
* [ ] 升级 Fabric Adapter，适配全新的 DPKI 查询接口。

### Phase 3: L1 层双阶段存证与 DAG 并行优化 (L1 Smart Contract)
* [ ] 编写 FISCO BCOS `CrossDomainAuth.sol` 合约。
* [ ] 实现**“请求-响应”双阶段逻辑**：引入 `mapping(uint256 => RequestRecord)`，阶段一记录请求，阶段二通过预言机回调完善哈希与授权回执。
* [ ] **DAG 并行优化**: 在 Solidity 合约中引入 FISCO BCOS 的并行框架，消除状态读写冲突，为高频通感数据上链开启并行交易打包支持。

### Phase 4: 共识引擎与信任域自动部署 (Deployment Automation)
* [ ] 开发 `baas_deploy.sh` 引擎脚本与 `topology.yaml`。
* [ ] 实现根据 yaml 配置，一键自动拉起 Fabric (Raft) 核心域，以及 FISCO BCOS 多群组 (Group1: PBFT, Group2: rPBFT)。
* [ ] 在 `auto_trigger.go` 中引入路由逻辑：基于信任域标识，自动分发预言机任务，并将结果回写至对应的共识群组。
